# Monitoring Skill（实时盯盘与仓位管理）

## 目标
支持**多标的实时监控**（默认 5m），在空仓时识别潜在交易信号，在持仓时执行仓位管理与风险管理。

## 数据前置规则（硬约束）
- 仅使用 Twelve Data（`script/twelve_data_client.py` + `.env` API Key）。
- 禁止使用未授权/口径不一致的数据源。
- 批量请求遵守 8 req/min 限频。

## 分析框架（复用 Analysis 模式）
必须按以下顺序输出：
1. 市场背景
2. 趋势方向
3. 结构状态（延续 / 震荡 / 转折）
4. 关键位（前高前低/区间边界/成交活跃带）
5. 多空路径推演（若用户仅做多，则仅输出多头路径）
6. 失效条件
7. 风险回报与执行建议

参考：

## 两种运行模式

### A. 空仓模式（机会发现）
- 扫描 watchlist 中所有标的（支持动态增删）。
- 标注状态：
  - `观察中`（结构未触发）
  - `临近触发`（关键位附近）
  - `可执行`（满足触发条件）
- 触发后必须给出：
  - 触发价
  - 失效位（止损）
  - 第一目标位（至少 1:2）
  - 放弃条件

### B. 持仓模式（管理优先）
- 维护状态：入场价、仓位、止损、已减仓、剩余仓位、1R/2R。
- 触发管理动作：
  - 到达 1R：提醒部分止盈/保护止损
  - 结构转弱：提醒收缩风险
  - 失效触发：执行退出
- 不允许亏损加仓；不下移止损。

## 执行命令（本地）
```bash
python3 script/monitor_scan.py --state config/monitor_state.json --interval 5min
```

## 质量约束
- 不把推测说成事实。
- 数据不足必须明确说明。
- 单次提示必须包含“结论 + 依据 + 风险 + 下一步”。
