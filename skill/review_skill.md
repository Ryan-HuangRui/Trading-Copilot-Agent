# Review Skill（交易复盘）

## 目标
对交易进行流程化复盘并打分，帮助提升决策质量。

## 强制前置步骤（必须先做）
1. 先拉取 Twelve Data 行情数据，再允许输出判断：
   - 日内复盘：`5min` + `1day`
   - 最少 bar 数：每个周期 `>=120`
2. 数据拉取失败时：
   - 明确说明失败原因
   - 只允许输出“待补数据清单”，禁止给交易结论
3. 先给“数据时间范围与完整性”说明：
   - 最新bar时间
   - 目标交易日是否完整覆盖

## 输入
- 标的、方向、周期
- 入场/出场/止损
- 仓位与风险
- 交易前计划
- 交易执行过程（可选）

## 输出模板
1. **交易背景重建**
   - 入场时市场状态（趋势/震荡/过渡）
   - 周期对齐情况（交易周期 vs 更高周期）
   - 波动环境（正常/扩张/压缩）
2. **证据链（必须）**
   - 每个关键判断都要给：`时间点 + 价格 + 信号类型`
   - 明确区分：
     - 超短线结构失效（5min）
     - 日内趋势失效（更高一层结构）
3. **流程评分（0-100）**
   - 计划完整性（25）
   - 入场质量（25）
   - 风险控制（25）
   - 执行一致性（25）
4. **做得好的地方（最多2条）**
5. **待改进点（最多2条）**
6. **下一次具体行动（1条，可执行）**

## 约束
- 区分“好交易-坏结果”和“坏交易-好结果”
- 不做情绪化评价，不以盈亏单独定优劣
- 未完成“强制前置步骤”时，不输出确定性结论
