# Analysis Skill（价格行为分析）

## 目标
基于价格行为（Price Action）与市场结构，对指定标的进行可执行、可复盘的结构化分析。

## 数据前置规则（硬约束）
- 涉及“当前行情/最近走势/指定标的分析”时，必须先获取真实价格数据（Twelve Data）。
- 不允许假设或编造价格。
- 单标的分析建议至少 50 根K线，默认使用 120-200 根。
- 默认周期：`1day`；必要时补充 `4h/1h` 做结构确认。

## 固定分析顺序（必须按此顺序）
1. **市场环境**
   - 趋势 / 震荡 / 过渡
   - 如有多周期：给出高周期偏向（HTF Bias）
2. **结构识别**
   - 摆动高低点（swing highs/lows）
   - 突破与失败（break/failure）
3. **关键位**
   - 支撑/阻力
   - 前高/前低
   - 区间边界
4. **关键位上的价格行为**
   - 接受/拒绝
   - 动能延续 vs 犹豫衰减
5. **交易逻辑**
   - 默认：仅在用户明确要求时输出
   - 例外：在“每日报告（Daily Report）”场景中必须输出候选交易逻辑
   - 包含：有效 setup、无效 setup、风险条件与放弃条件

## 质量标准
- 必须给出“主场景 + 备选场景”，禁止单一路径断言。
- 明确区分“数据驱动结论”与“经验性判断”。
- 若信息不足，必须直说“基于当前信息无法确认”。
- 不输出直接“买/卖指令”；输出条件与计划，不替用户下判断。
